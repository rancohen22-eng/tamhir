# ריכוז הלוואות · מחלקת חשבות (ארקיע)

אתר עצמאי (קובץ HTML יחיד) לריכוז לוחות הסילוקין ופרטי ההלוואות, במיתוג ארקיע.

## מה יש במערכת
- **מסך התחברות** (שם משתמש + סיסמה). הנתונים נשמרים **מוצפנים** (AES‑GCM) ב‑localStorage של הדפדפן; הסיסמה נגזרת (PBKDF2) ומפענחת אותם — בלי סיסמה נכונה אין גישה לנתונים.
- **דשבורד**: כרטיסי סיכום (סך קרן מקורית, יתרת קרן ליום נבחר, חלות קרן/ריבית ל‑12 חודשים), וטבלת הלוואות עם מטבע, בנק מממן, ריבית, יתרה ליום, חלות שנתית ותאריך סיום. לחיצה על שורה פותחת פירוט מלא + **לוח סילוקין**.
- **בורר "יתרה ליום"**: מחשב את יתרת הקרן בכל תאריך.
- **מסך פרעונות**: ריכוז לפי חיתוך **חודש / רבעון / שנה**, עם הפרדה בין **קרן / ריבית / קרן+ריבית**, בחירת טווח שנים ומטבע, וייצוא לאקסל (CSV).
- **מנוע לוח סילוקין**: מחשב אוטומטית שפיצר (תשלום קבוע), קרן שווה, ובוליט/בלון, כולל גרייס קרן. ניתן גם להדביק/לייבא לוח ידני שדורס את החישוב.
- **שערים**: שער יציג נמשך מ‑API של **בנק ישראל** (best‑effort בדפדפן), עם עדכון ידני כגיבוי אמין. המרות לשקל (או למטבע בסיס אחר) לפי השער ליום.
- **גיבוי**: ייצוא/ייבוא קובץ מוצפן להעברה בין מכשירים (נדרשת אותה סיסמה).

## פרטי התחברות (ברירת מחדל זמנית)
```
שם משתמש: hashavut
סיסמה:    Arkia2026!
```
> יש להחליף לפני שימוש אמיתי.

## החלפת שם משתמש / סיסמה
הנתונים מוצפנים במפתח שנגזר מהסיסמה, לכן שינוי הסיסמה נעשה ע"י יצירת בלוק הרשאות חדש:
```bash
cd loans
node rekey.mjs <username> <password>            # משתמש יחיד
node rekey.mjs user1 pass1 user2 pass2          # כמה משתמשים (חולקים אותם נתונים)
```
העתיקו את בלוק `const USERS = [...]` שהתקבל, והחליפו בו את הבלוק ב‑`index.html` (חפשו `const USERS`).
כל המשתמשים שנוצרים בהרצה אחת חולקים מפתח נתונים משותף.

## הערות
- הכל סטטי — אין שרת. ניתן לפרוס ב‑GitHub Pages וכד' (הכתובת תהיה `/loans/`).
- שערי בנק ישראל נמשכים ישירות מהדפדפן; אם סביבת הרשת חוסמת זאת (CORS), עדכנו שערים ידנית — שאר המערכת פועלת כרגיל.
- הלוואות בריבית משתנה (פריים/מדד) מוצגות בהיטל לפי הריבית הנוכחית שהוזנה; ללוח מדויק אפשר להדביק לוח ידני.
