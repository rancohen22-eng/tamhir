# מערכת תיעוד הזמנות טיסות · ארקיע

מערכת ענן לניהול ותיעוד מחזור החיים של הזמנות טיסה בארגון — מהייזום, דרך קבלת הצעת מחיר מהסוכן ואישור הגורם המאשר, ועד לכרטוס ותיעוד המסמכים. במיתוג ארקיע, דו-לשונית (עברית/אנגלית), עם התראות, מעקב מלא ודשבורד כספים.

בנויה על **Oracle APEX + Oracle Autonomous Database** (חינמי, Always Free; APEX/ORDS כלולים ללא רישיון נפרד), על אותה סביבת אורקל שבה כבר רצה מערכת התמחיר.

> **שים לב:** תיקייה זו מכילה את **שכבת בסיס הנתונים המלאה** (DDL, PL/SQL, seed, Views) + **מדריך בנייה** של אפליקציית ה-APEX. עמודי ה-APEX עצמם נבנים ב-APEX Builder לפי [`apex/BUILD_GUIDE.md`](apex/BUILD_GUIDE.md) — זהו אופי העבודה ב-APEX (הגדרת עמודים ב-GUI מעל הסכמה).

---

## יכולות עיקריות
- **מחזור חיים מלא של הזמנה:** ייזום → הצעת מחיר מהסוכן → אישור → כרטוס, כולל **ביטול דו-שלבי** (בקשה + אישור הסוכן).
- **מינימום הזנה:** המזמין פותח בקשה עם מעט שדות; מחלקה, גורם מאשר, תאריך פתיחה ומטבע (USD) נגזרים אוטומטית.
- **תפקידים והרשאות:** מנהל מערכת, מזמין, גורם מאשר (לכל מחלקה), סוכן, כספים. משתמש שייך למחלקה אחת אך יכול לשאת כמה תפקידים.
- **סוכן חיצוני:** לסוכן חשבון במערכת; הוא רואה רק בקשות שהופנו אליו, מזין מחיר/פרטי נסיעה, מעלה צילום ממערכת ההזמנות, ומכרטס.
- **מסמכים:** צילום ההצעה + מסמך/תמונת הכרטוס הסופי נשמרים כ-BLOB בבסיס הנתונים.
- **התראות:** פעמון בתוך המערכת (מונה לא-נקראו) **וגם** אימייל (APEX_MAIL) — לכל נמען בשפת ההעדפה שלו.
- **מעקב "מי עשה מה ומתי":** בפתיחת הזמנה רואים ציר זמן של כל הסטטוסים הקודמים, מבצע ותאריך.
- **דשבורד כספים + ייצוא Excel** ו**מחולל טמפלט דוחות** בבחירת שדות.
- **דו-לשוני:** ברירת מחדל עברית (RTL), החלפה לאנגלית (LTR) במתג דגל 🇮🇱/🇺🇸; גם הסטטוסים בעברית.
- **מיתוג:** ארקיע כמותג-על + לוגו הסוכן (טוסטוס) ב-co-branding במסכים ובמסמכים הרלוונטיים לסוכן.

## תפקידים
| תפקיד | code | הרשאות עיקריות |
|---|---|---|
| מנהל מערכת | `ADMIN` | ניהול מחלקות, משתמשים, מאשרים, סוכנים, מטבעות |
| מזמין | `INITIATOR` | פתיחת הזמנה, הפניה לסוכן, בקשת ביטול, צפייה בהזמנות המחלקה |
| גורם מאשר | `APPROVER` | אישור/דחייה, בקשת ביטול |
| סוכן | `AGENT` | הצעת מחיר, כרטוס, אישור ביטול — רק לבקשות שהופנו אליו |
| כספים | `FINANCE` | דשבורד כספים, ייצוא Excel, טמפלטי דוחות |

## מחזור חיים (סטטוסים)
`NEW → AWAITING_QUOTE → QUOTE_RECEIVED → APPROVED → TICKETED`
ענפים: `REJECTED` · `CANCEL_REQUESTED → CANCELLED`

| מעבר | פעולה | מקבלי התראה |
|---|---|---|
| → AWAITING_QUOTE | המזמין מפנה לסוכן | הסוכן |
| → QUOTE_RECEIVED | הסוכן מגיש הצעה + צילום | המזמין |
| → APPROVED | המאשר מאשר | הסוכן + המזמין |
| → REJECTED | המאשר דוחה | המזמין + הסוכן |
| → TICKETED | הסוכן מכרטס (PNR + מזהה חד-ערכי + מסמך) | המזמין + המאשר |
| → CANCEL_REQUESTED | המזמין/המאשר מבקש ביטול | הסוכן + המאשר |
| → CANCELLED | הסוכן מאשר ביטול | המזמין + המאשר |

## מבנה התיקייה
```
bookings/
├── README.md                ← מסמך זה
├── db/
│   ├── 01_tables.sql        ← DDL: טבלאות, מפתחות, אינדקסים
│   ├── 02_seed.sql          ← נתוני יסוד: מחלקות, מטבעות, תפקידים, סטטוסים, תבניות התראה, משתמשי דמו, טוסטוס
│   ├── 03_booking_pkg.sql   ← חבילת workflow: change_status + עוטפי מעברים + אימות
│   ├── 04_views.sql         ← Views: הזמנות, ציר זמן, סיכום כספים (דו-לשוני)
│   └── 05_grants.sql        ← הרשאות/ORDS (הערות — להריץ רק את הרלוונטי)
├── apex/
│   └── BUILD_GUIDE.md       ← מדריך שלב-אחר-שלב לבניית אפליקציית ה-APEX
└── assets/
    ├── arkia-logo.svg       ← לוגו ארקיע (מותג-על)
    └── toustous-logo.jpeg   ← לוגו הסוכן טוסטוס (co-branding)
```

## אבטחה
- סיסמאות נשמרות כ-hash (SHA-256) עם salt ייחודי לכל משתמש (`STANDARD_HASH`, ללא סיסמאות בטקסט גלוי).
- הרשאות נאכפות ב-APEX Authorization Schemes לכל תפקיד; הסוכן מוגבל לבקשות שהופנו אליו בלבד.
- קבצים נשמרים כ-BLOB בבסיס הנתונים (גיבוי אחיד עם שאר הנתונים).
- לפני עלייה לאוויר: החליפו סיסמאות דמו, הגדירו כתובת שולח מייל אמיתית (`booking_pkg.g_mail_from`), והגבילו גישת רשת לפי מדיניות הארגון.
