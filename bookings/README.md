# מערכת תיעוד הזמנות טיסות · ארקיע

מערכת ענן **רב-משתמשית** לניהול ותיעוד מחזור החיים של הזמנות טיסה בארגון — מהייזום, דרך הצעת מחיר מהסוכן ואישור הגורם המאשר, ועד לכרטוס ותיעוד המסמכים. במיתוג ארקיע, דו-לשונית (עברית/אנגלית), עם התראות, מעקב מלא ומסך כספים.

רצה על **Oracle Autonomous Database** (Always Free): סכמה ייעודית (`ARKIA`) עם לוגיקת PL/SQL, שכבת **REST דרך ORDS**, ואפליקציית ווב (SPA) המוגשת ישירות מ-ORDS. פריסה **אוטומטית** (CI/CD) בכל שינוי.

## 🔗 גישה
- **האפליקציה:** `/ords/arkia/api/app` (על מארח ה-ADB שלכם).
- **התחברות דמו** (סיסמה `Arkia2026!` — לשנות!): `admin` · `initiator_acmi` · `approver_acmi` · `finance` · `teltos` (סוכן).

## יכולות
- מחזור חיים מלא: ייזום → הצעת מחיר (+צילום) → אישור/דחייה → כרטוס (PNR+מזהה+מסמך), כולל **ביטול דו-שלבי**.
- **תפקידים** ואכיפה בשרת: אדמין/מזמין/מאשר/סוכן/כספים; הסוכן רואה רק בקשות שהופנו אליו, מזמין/מאשר — הזמנות המחלקה.
- **המאשר** יכול גם לפתוח הזמנה חדשה וגם **לערוך** הזמנה (עד שלב הכרטוס), בנוסף למזמין ולאדמין.
- **פרטי הזמנה מורחבים**: מספר נוסעים, שעה מועדפת (רשות), **נתיב טיסה לפי קוד IATA** (מוצא/יעד מטבלת שדות-תעופה), ו**מפעיל רטוב** למחלקת ACMI (טבלה שמנוהלת ע"י אדמין). בייזום אין עדיין מחיר — הוא נקבע ע"י הסוכן בהצעה.
- **דשבורד** בראש מסך ההזמנות (סיכומי סטטוסים) ועמודת "עודכן לאחרונה ע"י".
- **קבצים** כ-BLOB (העלאה/הורדה), **פעמון התראות** (מונה לא-נקראו), **ציר-זמן** "מי/מתי/מה".
- **מסך כספים**: **סינון לפני הפקה** (טווח תאריכי כרטוס, מחלקה, סטטוס), **בורר עמודות לייצוא** ו**תבניות דוח** (נשמרות לדפדפן), ייצוא CSV/Excel.
- **מסכי ניהול**: מחלקות, משתמשים (הצגת אימייל, **עריכה**, הפעלה/השבתה, **איפוס סיסמה**), מפעילים רטובים, סוכנים ומטבעות — הכל באפליקציה, ללא SQL.
  - **סיסמאות** נשמרות כ-hash חד-כיווני ולכן **לא ניתן להציגן** לאיש (גם לא לאדמין). איפוס יוצר סיסמה זמנית המוצגת פעם אחת; כשיופעל שירות המייל אפשר יהיה לשלוח קישור איפוס.
- **דו-לשוני** עברית/אנגלית עם החלפת דגל 🇮🇱/🇺🇸 (RTL/LTR).
- **מיתוג** ארקיע + לוגו הסוכן טלטוס (co-branding).
- **התראות מייל** — דרך **OCI Email Delivery** (SMTP · STARTTLS 587). הגדרה חד-פעמית: `deploy/email_setup.sql` (ADMIN) + בדיקה `deploy/email_test.sql`. השולח: Approved Sender מאושר ב-OCI.

## התקנה (תמצית)
1. **בסיס נתונים** (פעם אחת, כ-`ARKIA`): הריצו את `db/install_all.sql`.
2. **REST + UI**: הריצו `api/install_api.sql` ואז `api/install_ui.sql`.
3. **גישה**: פתחו `/ords/arkia/api/app`.
4. **פריסה אוטומטית**: לאחר הגדרת ה-CI (ראו `deploy/README.md`), כל push לקבצי `api/**`/`app/**` מתקין את השינוי אוטומטית — ללא הרצה ידנית.

> הערה: `bookings/index.html` (בשורש התיקייה) הוא **גרסת MVP** בקובץ יחיד עם `localStorage` — זהה בפשטות למערכות הסטטיות, אך **ללא שיתוף בין משתמשים/מכשירים**. הגרסה הרב-משתמשית היא זו שב-`app/` + `api/`.
